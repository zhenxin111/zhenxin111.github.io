适配器

知识点：
适配器模式（Adapter）是JS设计模式的一种。适配器就像一台接口转换机，不过它并非完全改变一个接口，它只是创建一个新的对象和函数，去适配现有对象或函数的接口，去兼容已存在在使用的代码。

带例子的参考：JavaScript设计模式之适配器模式

JS中this的指向

选项(选择正确的一项)：

A 在函数定义时，this指向全局变量
B 在浏览器下的全局范围内，this指向全局对象
C 当对象调用函数或者方法时，this指向这个对象
D 在使用new实例化对象时，this指向这个实例对象

选项有歧义，一眼看过去两三个选项都像对的，注意文字陷阱；A的全局变量名词错误；B的全局范围用词不准确，在任何函数体外部的全局范围？
知识点：

this指向的是，调用函数的那个对象。

分以下四种情况详细讨论this用法：

情况一：直接函数调用

非严格模式，this指向全局对象

严格模式下，this未被执行的上下文环境定义，它默认为undefined

情况二：作为对象方法的调用

函数作为方法被对象调用，this指向调用该函数的对象

情况三：作为构造函数调用

如果函数倾向于和new关键词一块使用，则我们称这个函数是构造函数。在函数内部，this 指向新创建的对象。

情况四：call 和 apply调用

call()函数和apply()函数是每个函数都有的方法，可以通过设置第一个参数，改变绑定到指定对象上；
apply()方法接受两个参数：第一个参数就表示改变后的调用这个函数的对象，第二个是参数数组（可以是Array的实例，也可以是arguments对象）
call()方法和apply()方法的区别在于接受参数的方式不同，对于call()方法来说，第一个参数不变，变化的是后面的参数必须逐个列举出来；

编程题（3题，一题20分）

漂亮瓷砖

由一串字母字符串进行排列组合，得到的多个字符串中，只有一对不同字母相邻的排列字符串的个数；例如：s=’AABB’;有6中排列可能，其中的”AABB”和”BBAA”满足要求
输入:

字符串s，字符串长度length(1~50),s中字符均为大写字母

输出:

一个整数，表示可以有的多少种方式

01串的子串

01串表示由0和1作为字符组成的字符串，交错01串表示任意两个相邻位置的字符都不一样的01字符串；求一段01串中最长的交错01子串；
输入

01字符串s，s的长度lenth(1~50)

输出：

一个整数，表示最长01子串的长度

独立日

小易每天要吃一个水果并且交X元房租，他手上有f个水果和d元，店里一个水果p元；小易能活几天？
输入：

一行4个整数，空格隔开:x f d p(x>=1,f,d,p<=2*10^9)

输出：

一个整数，表示小易活的天数
